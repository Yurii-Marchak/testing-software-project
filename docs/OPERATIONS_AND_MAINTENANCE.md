# Супровід і експлуатація

## 1. Конфігурація середовища

Застосунок читає параметри з `.env` або змінних оточення.

### Обов’язкові змінні

- `DB_PASSWORD`

### Необов’язкові змінні

- `DB_HOST`
- `DB_USER`
- `DB_NAME`
- `APP_SECRET_KEY`
- `APP_HOST`
- `APP_PORT`
- `FLASK_DEBUG`

## 2. Локальний запуск

```powershell
py main.py
```

Примітка: при такому запуску використовується Flask development server. Це нормально для локальної розробки, але не для production.

## 3. Docker-запуск

```powershell
docker-compose up --build
```

## 4. Підготовка БД

```powershell
mysql -u root -p < init_db.sql
```

Скрипт:
- створює БД `lab7`;
- створює таблиці;
- додає тестові дані;
- створює тригери для `PC_Build.total_price`.

## 6. Типові проблеми

### 6.1. Не задано пароль до БД

Причина:
- `DB_PASSWORD` відсутній у `.env` або змінних середовища.

Рішення:
- перевірити `.env`;
- перевірити лапки в паролі;
- переконатися, що пароль не порожній.

### 6.2. Помилка доступу до MySQL

Можливі причини:
- неправильний пароль;
- MySQL не запущений;
- неправильне ім’я БД;
- немає користувача або прав.

### 6.3. Попередження Flask про development server

Повідомлення виду:

```text
WARNING: This is a development server. Do not use it in a production deployment.
```

Це не помилка. Воно означає, що локально застосунок запущено на dev server.

### 6.4. `pytest` не створює `.pytest_cache`

У цьому середовищі можуть з’являтися попередження про відсутність доступу до кеш-каталогу.

Вплив:
- тести все одно можуть проходити успішно;
- це не означає, що код зламаний.
